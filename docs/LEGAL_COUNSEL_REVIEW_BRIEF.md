## 4. 規約・ポリシーで採用している主な方針（要レビュー）

### 4.1 準拠法・裁判管轄（利用規約）

- **日本法**を準拠法とし、**東京を管轄**とする旨を記載。
- 利用者の居住国の**消費者保護などの強行法規**が優先しうる旨の但し書きあり。
- 旧版にあった**カリフォルニア法・個別仲裁・集団訴訟放棄**は削除・置換済み（米国向け条項の再設計が必要かは別途判断）。

### 4.2 英国（UK）

- 利用規約末尾に **UK 向け Addendum**（UK GDPR / DPA 2018、消費者権、ICO、Online Safety Act の**論点言及**）。
- プライバシーに **Article 8b（UK）** を追加。
- **Online Safety Act の適用該当性**は条文上「事業実態次第」とし、**英弁護士による適用判断**を推奨するスタンス。

### 4.3 米国（DMCA）

- アプリ内に **DMCA Policy 画面**を新設。Designated Agent の連絡先に上記 Gmail を記載。
- 同一画面に **英国向けの短い説明**（英国の著作権制度は別であり、同じ連絡先で UK 法に照らして評価する旨）を追記。
- **著作権局へのエージェント登録**の有無・登録内容と文面の一致は、米国側実務で確認が必要。

### 4.3b コミュニティガイドライン

- `app/community-guidelines.tsx` に、尊重・違法コンテンツ禁止・ライブ・通報・執行の**一般方針**を英語で記載。
- 利用規約の禁止行為・ライブ条項から **当該ページへリンク**。設定画面・ホーム・未ログインプロフィールのフッターからも参照可能。

### 4.4 コンテンツライセンス（クリエイター投稿）

- 旧「マーケティング含む広いライセンス」から、**サービス運営・改善および RawStock 上・関連プロモの範囲**に**目的を絞る**方向で文言調整。
- 域外マーケ・再ライセンスは**別オプション**の余地を文面に残している。**クリエイター契約・レーベル**との整合は要確認。

### 4.5 ライブ・UGC

- ライブ・リアルタイム機能に関する**モデレーション・通報・停止**の方針を規約に追記（具体的 SLA は「状況により異なる」程度）。

### 4.6 未成年

- **13 歳未満は対象外**、13–17 は**保護者同意** — Terms と Privacy で整合。

### 4.7 責任制限

- 上限を **「過去12か月の支払額」と「10,000 円」のいずれか高い方**等に変更。**強行法規での無効**の可能性は各法域で要検討。

### 4.8 プライバシー — サブプロセッサ・データの例示

文書上、例として列挙している処理者のカテゴリには、**Neon、Cloudflare、Google、Stripe、Upstash、Anthropic（モデレーション等）、DeepAR（カメラ・顔トラッキング／AR）** 等を記載。  
**実装と完全一致しているか**はエンジニア確認・データマップ整備が望ましい。

### 4.9 削除・保持（プライバシー）

- アカウント削除後の削除目安（**おおむね30日**）に加え、**税務・紛争・バックアップ・コミュニティオーナー**等の**例外**を記載。  
- アプリの削除 API は**コミュニティオーナーは削除不可**等の実装あり — **文書と実装の一致**を維持する必要あり。
- **設定画面のアカウント削除確認ダイアログ**でも、コミュニティ所有の制限・法的保持・バックアップについて **Privacy Policy と整合する旨**を短文で表示。

### 4.10 クリエイター払い出し・Stripe

- **Stripe Connect オンボード前**に、規約（報酬・税務自己責任・Stripe 法務）の**要約とチェックボックス**を表示。
- サーバーに **`payout_terms_agreed_at`** を記録する API を追加。**オンボード API は未同意時 400**。
- 本番 DB には **`migrations/0012_users_payout_terms_agreed_at.sql`** の適用が必要。

## 5. 弁護士に特に見てほしい論点（チェックリスト）

1. **契約当事者・準拠法・管轄**（日本個人事業主＋グローバル利用者）の書き分けは十分か。EU / UK / 米国の**消費者**に対する効力。
2. **仲裁・クラスアクション放棄を外した**ことによる米国リスクと、代替条項の要否。
3. **UK Online Safety Act** の適用有無と、該当する場合の**ポリシー・プロセス・年齢保証**の充足。
4. **DMCA** エージェント表示と**実際の通知対応フロー**（社内オペレーション含む）の整合。
5. **DeepAR 等**の生体情報・顔映像に近いデータの**開示の十分性**（米州法 BIPA 等を含むリスク整理）。
6. **特商法**: `legal-notice` と `tokusho` の**二重表示の要否・差分**、電話番号非掲載の説明で足りるか。
7. **クリエイター報酬・源泉・インボイス・国外クリエイター**の条文・画面説明の十分性。
8. 英語のみの規約で**日本ユーザー**に対する説明・同意の取り方（日本語版の要否）。
